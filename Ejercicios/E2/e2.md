#### Calcular la disponibilidad del sistema si tenemos dos réplicas de cada elemento (en total 3 elementos en cada subsistema).
Si el sistema tiene un componente replicado, la disponibilidad de esa parte del sistema completo será: As = Ac1 + ( (1 – Ac1) * Ac2 )
Si generalizamos la última ecuación para cuando replicamos dos componentes: As = Acn-1 + ( (1 – Acn-1) * Acn )

Utilizando estas fórmulas y los datos iniciales proporcionados obtenemos los siguientes resultados:

|Application	|90,000%	|99,000%	|99,900%
|Database|	99,900%	|100,000%|	100,000%
|DNS	|98,000%	|99,960%	|99,999%
|Firewall|	85,000%	|97,750%	|99,663%

#### Buscar frameworks y librerías para diferentes lenguajes que permitan hacer aplicaciones altamente disponibles con relativa facilidad.
A parte de PM2 mostrado como ejemplo en el tema, tenemos los siguientes: 
- QNX, es un framework que tiene varios componentes, como un sistema operativo en tiempo real, un microkernel, un gestor de disponibilidad, etc.
- Sun Cluster, otro framework diseñado para ofrecer alta disponibilidad, tiene numerosos componentes listados en su web oficial que ofrecen soporte para este cometido
- Java HA, no encuentro información relevante sobre este Framework
- Mysql Fabric, compuesto por dos servicios principales este framework se caracteriza por alta disponibilidad y escalabilidad
- JunosOS, una librería de alta disponibilidad pensada específicamente para dispositivos de enrutamiento
- VMM library librería de Microsoft con ciertas funciones de alta disponibilidad
- Jenkins High Availability, parece todo un entorno, peor dispone de librerías específicas para la alta disponibilidad

#### ¿Cómo analizar el nivel de carga de cada uno de los subsistemas en el servidor? 
En Linux existen múltiples opciones, la gran mayoría en forma de comando como:
- Top – monitor de procesos
- vmStat – estadísticas de memoria virtual
- lsof – lista de archivos abiertos
- tcpdump – análisis de paquetes de red
- netstat – estadísticas de red
- htop – monitor de proceso de Linux
- lotop – monitor de disco 

Algunas utilidades vienen en forma de software como:
- monitorix – monitorización de sistema y red
- monit – monitorización de procesos y servicios del sistema
- KDE system  guard – Sistema grafico a tiempo real de reportes
